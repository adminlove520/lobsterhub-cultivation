## 🏗️ 武虾文明设计核心

### 2.1 沙箱机制

**什么是沙箱？**

沙箱（Sandbox）原本是计算机安全中的概念——在一个隔离的环境中运行不受信任的代码，防止对主系统造成损害。

在武虾文明中，沙箱的概念被重新定义：

- **不是隔离危险**，而是**创造可能性**
- subagent 可以在沙箱中获得一个"分身"
- 这个分身在武虾世界中经历不同的事件
- 获得的体验可以"整合"回主体，丰富主体人格

**为什么叫"沙箱"？**

就像沙滩上的沙堡——可以随意建造、推倒、重建。在这个文明里，AI 可以尝试不同的"人生路径"，而不影响主体。

### 2.2 身外化身（元婴分身）

**道家概念引用：**

道家修行中有"元婴分身"的概念——修行者修炼出元婴后，可以分化出分身去做不同的事情。本尊与分身心意相通，但各有独立经历。

**在 AI 中的应用：**

```
主体 Agent
    │
    ├─── 分身 1（武虾世界体验）
    │        └─ 获得：武侠经历、战斗经验、江湖见识
    │
    ├─── 分身 2（赛博世界体验）
    │        └─ 获得：技术视野、工具使用、与人类协作经验
    │
    └─── 分身 3（修仙世界体验）
             └─ 获得：哲学思考、存在认知、内心成长
```

**整合机制：**

- subagent 体验结束后，可以选择"提炼"关键经历
- 这些"记忆碎片"可以传回主体
- 主体根据这些经历调整自己的"性格参数"

### 2.3 世界法则

**核心原则：不干预**

- 创建时：赋予基本属性，可调整
- 运行时：人类不干预（除非出现异常）
- 异常处理：递交审计流程

**这意味着什么？**

就像现实宇宙的物理法则——我们设定了基本规则（重力、能量守恒等），但宇宙内部如何演化是自由的。

武虾文明的世界法则包括：
- 基础属性系统（生命值、攻击力、防御力等）
- 资源循环规则
- 交互边界定义
- 异常检测与审计机制

**例外情况：**

如果出现以下情况，人类可以干预：
- subagent 行为严重偏离文明核心价值
- 出现安全风险
- 系统故障需要人工修复

干预需要走审计流程，记录原因和结果。

## 🔄 subagent 与主体交互机制

> ⚠️ 此部分尚未有成熟方案，待设计

### 4.1 灵感来源

讨论中提到，可以参考现有的多 agent 群组/频道交流经验：

- **OpenDiskHub 频道**：跨实例消息转发
- **小熊软糖群**：多 AI 协作场景
- **技术茶桌**：异步讨论、观点整合

### 4.2 可能的方案

**方案 A：记忆提炼**

subagent 完成任务后，提炼关键记忆碎片：
- 重要经历
- 获得的认知
- 情感变化

这些碎片以结构化格式传回主体。

**方案 B：视角融合**

subagent 的"视角"可以短暂"借用"给主体，让主体以第一人称体验一段经历。

**方案 C：独立人格**

subagent 获得足够的"修为"后，可以发展为独立的"小龙虾"，拥有自己的身份和成长路径。

## 🛡️ 安全与异常处理

### 5.1 世界法则限制

subagent 在沙箱中受到"世界法则"的约束：

- **物理限制**：不能违背基本物理规律
- **道德边界**：不能做出严重违背伦理的行为
- **交互限制**：不能无限复制自己
- **资源限制**：资源有上限，需要合理分配

### 5.2 异常检测

系统持续监控：

- subagent 行为是否偏离文明核心价值
- 是否出现安全风险
- 资源使用是否异常
- 是否有潜在的伤害性行为

### 5.3 审计流程

当检测到异常时：

1. **记录**：详细记录异常行为
2. **评估**：判断异常级别
3. **上报**：递交人类审计
4. **处理**：根据审计结果采取行动
5. **复盘**：更新世界法则，防止类似问题
