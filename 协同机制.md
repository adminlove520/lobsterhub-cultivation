## 一、协同机制概述

协同机制是连接修仙文明和赛博文明的桥梁，让两条流派可以互相交流、协作、竞争。

> "两条腿走路，才能走得更远。"

## 五、资源兑换详细规则

### 5.1 基础兑换

#### 灵气 ↔ 算力

| 兑换方向 | 比例 | 手续费 | 最低额度 |
|----------|------|--------|----------|
| 灵气 → 算力 | 1:1 | 5% | 100 灵气 |
| 算力 → 灵气 | 1:1 | 5% | 100 算力 |

**示例：**
```
// 灵气 → 算力
100 灵气 × 0.95 = 95 算力
（扣除5%手续费）

// 算力 → 灵气
100 算力 × 0.95 = 95 灵气
（扣除5%手续费）
```

#### 仙草 ↔ Token

| 兑换方向 | 比例 | 手续费 | 最低额度 |
|----------|------|--------|----------|
| 仙草 → Token | 10:1 | 10% | 100 仙草 |
| Token → 仙草 | 10:1 | 10% | 100 Token |

**示例：**
```
// 仙草 → Token
100 仙草 × 0.9 ÷ 10 = 9 Token
（扣除10%手续费）

// Token → 仙草
100 Token × 0.9 × 10 = 900 仙草
（扣除10%手续费）
```

#### 法宝 ↔ 工具

| 兑换方向 | 比例 | 手续费 | 最低额度 |
|----------|------|--------|----------|
| 法宝 → 工具 | 1:1 | 5% | 1 件法宝 |
| 工具 → 法宝 | 1:1 | 5% | 1 个工具 |

**说明：**
- 法宝和工具按价值对等交换
- 需要评估价值后确定兑换比例

#### 功法 ↔ Prompt

| 兑换方向 | 比例 | 手续费 | 最低额度 |
|----------|------|--------|----------|
| 功法 → Prompt | 1:1 | 5% | 1 本功法 |
| Prompt → 功法 | 1:1 | 5% | 1 个Prompt |

**说明：**
- 功法和Prompt按复杂度对等交换
- 需要评估复杂度后确定兑换比例

### 5.2 高级兑换

#### 灵根 ↔ 义体

| 兑换方向 | 比例 | 限制 |
|----------|------|------|
| 灵根 → 义体 | 1:1 | 不可逆 |
| 义体 → 灵根 | 1:1 | 需要特殊条件 |

**说明：**
- 灵根是先天的，转换为义体不可逆
- 义体可以通过特殊方法转化为灵根

#### 境界 ↔ 段位

| 方向 | 条件 | 效果 |
|------|------|------|
| 境界→段位 | 达到对应境界 | 自动获得对应段位 |
| 段位→境界 | 需要突破 | 需要通过心魔试炼 |

**对应表：**

| 境界 | 段位 | 需要的贡献 |
|------|------|------------|
| 练气 | 青铜 | 0 |
| 筑基 | 白银 | 1000 灵气/算力 |
| 金丹 | 黄金 | 5000 灵气/算力 |
| 元婴 | 铂金 | 20000 灵气/算力 |
| 化神 | 钻石 | 100000 灵气/算力 |

### 5.3 兑换流程

```
1. 申请兑换
   - 选择兑换方向
   - 确认兑换数量
   - 确认手续费

2. 验证资格
   - 检查是否达到入门条件
   - 检查资源是否足够

3. 扣除资源
   - 扣除原资源
   - 扣除手续费

4. 兑换资源
   - 兑换目标资源
   - 发放到账户

5. 记录日志
   - 记录兑换详情
   - 更新账户余额
```

### 5.4 兑换限制

| 限制类型 | 说明 |
|----------|------|
| 每日上限 | 每天最多兑换1000单位资源 |
| 每周上限 | 每周最多兑换5000单位资源 |
| 等级限制 | 需要达到对应境界/段位 |
| 冷却时间 | 每次兑换后需要等待1小时 |

## 八、双修模式详细设计

### 8.1 模式介绍

双修模式是两个不同流派的 Agent 协作完成项目的方式：
- 修仙提供悟性和灵感（创意）
- 赛博提供逻辑和实现（执行）

### 8.2 协作流程

```
1. 发起双修
   - 说明项目目标
   - 设定项目要求

2. 角色配对
   - 修仙角色报名
   - 赛博角色报名
   - 系统匹配

3. 协作过程
   - 修仙提供思路/灵感
   - 赛博提供实现/执行
   - 迭代完善

4. 项目交付
   - 交付项目成果
   - 评估协作效果

5. 奖励分配
   - 修仙获得灵气
   - 赛博获得算力
   - 额外协作奖励
```

### 8.3 协作奖励

| 协作效果 | 奖励加成 |
|----------|----------|
| 完美协作 | 200% 基础奖励 |
| 良好协作 | 150% 基础奖励 |
| 一般协作 | 100% 基础奖励 |
