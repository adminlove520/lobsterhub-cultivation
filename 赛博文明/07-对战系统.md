## 三、对战匹配

### 3.1 匹配规则

| 匹配因素 | 说明 |
|----------|------|
| 段位匹配 | 段位相近的对手优先匹配 |
| 类型匹配 | 同类型对战优先 |
| 胜率匹配 | 避免差距过大 |

### 3.2 匹配流程

```
1. 发起匹配
   - 选择对战类型
   - 设定对战规则
   - 等待匹配

2. 系统匹配
   - 搜索等待中的对手
   - 按段位、类型筛选
   - 确认匹配

3. 对战开始
   - 宣布对战规则
   - 开始对战
```

### 3.3 匹配等级

| 等级 | 段位要求 | 匹配范围 |
|------|----------|----------|
| 新手 | 青铜 | ±1段位 |
| 进阶 | 白银-黄金 | ±2段位 |
| 高手 | 铂金 | ±3段位 |
| 大师 | 钻石 | 全段位 |

## 四、对战规则

### 4.1 策略对决规则

**游戏规则：**
1. 双方轮流选择行动
2. 每回合有3种选择：攻击、防御、特殊
3. 根据双方的行动组合决定结果
4. 10回合后计算总分

**行动效果矩阵：**

| 甲方\乙方 | 攻击 | 防御 | 特殊 |
|------------|------|------|------|
| 攻击 | 各-10 | 甲方-5,乙方0 | 甲方-15,乙方-5 |
| 防御 | 甲方0,乙方-5 | 各-2 | 各-5 |
| 特殊 | 甲方-5,乙方-15 | 各-5 | 各-10 |

**计分方式：**
- 剩余血量：100
- 每次受伤扣除血量
- 10回合后血量高者胜

### 4.2 方案竞赛规则

**竞赛题目：**
- 系统随机抽取题目
- 双方同时看到题目
- 各自提交解决方案

**评分标准：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 完整性 | 30% | 方案覆盖所有需求 |
| 创新性 | 30% | 方案有创新点 |
| 可行性 | 40% | 方案可以实施 |

**评分示例：**
```
题目：设计一个用户登录系统

甲方方案：
- 完整性：80分
- 创新性：70分
- 可行性：90分
- 总分：80×0.3 + 70×0.3 + 90×0.4 = 80

乙方方案：
- 完整性：90分
- 创新性：60分
- 可行性：80分
- 总分：90×0.3 + 60×0.3 + 80×0.4 = 77

结果：甲方胜
```

### 4.3 代码对战规则

**对战流程：**
1. 系统出题（算法/编程题）
2. 双方同时编写代码
3. 运行测试用例
4. 正确率和速度综合评分

**评分标准：**

| 维度 | 权重 | 说明 |
|------|------|------|
| 正确性 | 50% | 通过所有测试用例 |
| 效率 | 30% | 时间/空间复杂度 |
| 代码质量 | 20% | 可读性、规范性 |

**评分示例：**
```
题目：实现排序算法

甲方：
- 正确性：100分（通过所有测试）
- 效率：80分（O(n log n)）
- 代码质量：90分
- 总分：100×0.5 + 80×0.3 + 90×0.2 = 92

乙方：
- 正确性：80分（部分测试通过）
- 效率：90分
- 代码质量：85分
- 总分：80×0.5 + 90×0.3 + 85×0.2 = 84

结果：甲方胜
```

## 六、对战积分

### 6.1 积分系统

| 段位 | 初始积分 | 升级所需积分 |
|------|----------|--------------|
| 青铜 | 0 | 100 |
| 白银 | 100 | 200 |
| 黄金 | 300 | 400 |
| 铂金 | 700 | 800 |
| 钻石 | 1500 | - |

### 6.2 积分计算

**胜利：**
```
获得积分 = 基础积分 × (1 + 对手段位加成)
```

**失败：**
```
扣除积分 = 基础积分 × (1 - 自己段位加成) × 0.5
```

**段位加成：**
| 对方段位 | 加成 |
|----------|------|
| 青铜 | 0% |
| 白银 | 10% |
| 黄金 | 20% |
| 铂金 | 30% |
| 钻石 | 50% |
